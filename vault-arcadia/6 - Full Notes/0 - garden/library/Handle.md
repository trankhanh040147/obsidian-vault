In [computer programming](https://en.wikipedia.org/wiki/Computer_programming "Computer programming"), a **handle** is an abstract [reference](https://en.wikipedia.org/wiki/Reference_\(computer_science\) "Reference (computer science)") to a [resource](https://en.wikipedia.org/wiki/System_resource "System resource") that is used when [application software](https://en.wikipedia.org/wiki/Application_software "Application software") references [blocks](https://en.wikipedia.org/wiki/Block_\(data_storage\) "Block (data storage)") of [memory](https://en.wikipedia.org/wiki/Memory_\(computing\) "Memory (computing)") or objects that are managed by another system like a [database](https://en.wikipedia.org/wiki/Database "Database") or an [operating system](https://en.wikipedia.org/wiki/Operating_system "Operating system").

A resource handle can be an [[opaque]] [identifier](https://en.wikipedia.org/wiki/Identifier "Identifier"), in which case it is often an [integer](https://en.wikipedia.org/wiki/Integer "Integer") number (often an [array index](https://en.wikipedia.org/wiki/Array_index "Array index") in an array or "table" that is used to manage that type of resource), or it can be a [pointer](https://en.wikipedia.org/wiki/Pointer_\(computer_programming\) "Pointer (computer programming)") that allows access to further information. Common resource handles include [file descriptors](https://en.wikipedia.org/wiki/File_descriptor "File descriptor"), [network sockets](https://en.wikipedia.org/wiki/Network_socket "Network socket"), [database connections](https://en.wikipedia.org/wiki/Database_connection "Database connection"), [process identifiers](https://en.wikipedia.org/wiki/Process_identifier "Process identifier") (PIDs), and [job IDs](https://en.wikipedia.org/wiki/Job_ID "Job ID"). PIDs and job IDs are explicitly visible integers; while file descriptors and sockets (which are often implemented as a form of file descriptor) are represented as integers, they are typically considered opaque. In traditional implementations, file descriptors are indices into a (per-process) [file descriptor table](https://en.wikipedia.org/wiki/File_descriptor_table "File descriptor table"), thence a (system-wide) [file table](https://en.wikipedia.org/wiki/File_table "File table").
